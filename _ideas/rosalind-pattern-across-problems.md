# The Patterns That Show Up in Every Kind of Problem

## The Idea
After solving 70 bioinformatics problems, certain approaches appear again and again: dynamic programming, graph traversal, string matching, combinatorics. The problems look different on the surface — one is about DNA, one is about sorting, one is about trees — but underneath, they're often the same pattern wearing a different costume. Recognizing that is the most valuable skill Project Rosalind teaches.

## Why It's Interesting
It's a story about how experts see the world differently from beginners. A beginner sees each problem as new and unique. An expert sees a new problem and asks "what does this remind me of?" That shift — from surface features to underlying structure — is transferable to almost any domain, from medicine to negotiation to software design.

## Key Points to Cover
- The observation: the same algorithmic patterns keep recurring across very different problems
- Examples: dynamic programming in sequence alignment AND coin change problems AND string edit distance
- What pattern recognition feels like when it kicks in: "I've seen this before"
- How this changes problem-solving speed: not starting from scratch every time
- The broader lesson: expertise is largely the ability to categorize new situations using patterns from old ones
