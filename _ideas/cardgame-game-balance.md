# Why Game Balance Is Harder Than It Looks

## The Idea
When you design a game with multiple pieces of equipment, you want each one to be useful — but not so useful that everyone always picks the same one. Achieving that balance is genuinely hard, and the usual approach (playing the game and tuning by feel) doesn't scale. Machine learning can discover which combinations are too powerful before human players do.

## Why It's Interesting
Game balance is a design problem disguised as a math problem. It shows up in games, but also in economics (tax policy), sports (salary caps), and software (pricing tiers). The challenge is always the same: create a system where multiple options are viable, none is dominant, and the choice between them is meaningful. Using AI to stress-test balance is a newer approach with broader implications.

## Key Points to Cover
- What "balance" means in a game: no single strategy that always wins
- Why balance is hard: equipment combinations interact in non-obvious ways
- The traditional approach: playtest, find dominant strategies, nerf them, repeat
- How running thousands of AI-vs-AI games finds balance problems faster
- What "emergent dominant strategies" are: combinations nobody designed to be powerful that turn out to be
